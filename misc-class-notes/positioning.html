CSS Positioning — Quick Guide

Four core values you’ll use most: static, relative, absolute, and fixed.

1) static (default)

  • Element stays in normal document flow.
  • top/left/right/bottom do nothing.
div {
  position: static; /* default */
}

2) relative

  • Like static, but you can nudge it.
  • Still takes up its original space.
.tag {
  position: relative;
  top: 10px;    /* moves down */
  left: 20px;   /* moves right */
}

3) absolute

  • Removed from normal flow.
  • Positions relative to the nearest positioned ancestor (relative/absolute/fixed); otherwise the page.
.badge {
  position: absolute;
  top: 8px;
  right: 8px;
}

4) fixed

  • Sticks to the viewport (browser window).
  • Doesn’t move when you scroll.
.fab {
  position: fixed;
  bottom: 16px;
  right: 16px;
}
static — normal flow, not movable
relative — in flow, can nudge
absolute — out of flow, inside nearest positioned ancestor
fixed — out of flow, stuck to viewport

Tip: To make an absolutely positioned child anchor to a parent, give the parent position: relative;.

Mini Examples

Absolute inside a relative parent

.parent { position: relative; }
.child  { position: absolute; top: 10px; right: 10px; }
Gray square sits at top-right of the parent.

Relative “nudge”

.chip { position: relative; top: 8px; left: 12px; }
Child moved, but its original space is still reserved.

Fixed button

.btn { position: fixed; bottom: 12px; right: 12px; }
Would stay put at the corner of the viewport while scrolling.
